# Синхронизация с WordPress

## Обзор

NaviBot получает актуальные цены из WordPress-сайта `teplohod-restoran.ru` через REST API. На сайте установлен PHP-сниппет (WPCode), который выгружает данные из ACF-полей товаров.

**Что синхронизируется:** только цены (сезоны, даты, время, дни недели, тариф).

**Что НЕ синхронизируется:** причалы, стоимость уборки, ссылки — редактируются вручную в админке (вкладка "Теплоходы").

## Архитектура

```
WordPress (teplohod-restoran.ru)
  └─ WPCode PHP snippet
      └─ REST endpoint: /wp-json/navibot/v1/prices
          │
          │  GET (JSON)
          ▼
NaviBot (app.py: sync_from_wp)
  └─ wp_parser.py
      ├─ parse_season_dates()  — русский текст → YYYY-MM-DD
      ├─ parse_time_field()    — "10.00 - 18.00" → "10:00", "18:00"
      └─ normalize_day_range() — "Пт  - Сб" → "Пт-Сб"
          │
          ▼
SQLite (prices table)
  └─ replace_prices_for_boat() — полная замена цен теплохода
```

## Формат данных из WordPress

```json
{
  "boats": [
    {
      "name": "Хемингуэй",
      "slug": "teplohod-hemingway",
      "prices": [
        {
          "season": "Низкий сезон",

          "time": "06.00 - 21.00",
          "day_range": "Вс - Чт",
          "price": 42000
        }
      ]
    }

```

## Парсер дат (wp_parser.py)

Главная сложность — текстовые описания дат на русском.

### Поддерживаемые форматы

| Входной текст | Результат |
|--------------|-----------|
| `до 14 мая и с 16 сентября` | [01-01 → 05-14], [09-16 → 12-31] |
| `с 15 мая по 9 июня и с 1 июля по 15 сентября` | [05-15 → 06-09], [07-01 → 09-15] |
| `с 10 июня по 30 июня` | [06-10 → 06-30] |
| `со 3 июля по 31 августа` | [07-03 → 08-31] |
| `от 10 сентября` | [09-10 → 12-31] |
| `по 2 июля` | [01-01 → 07-02] |
| `1 июля по 31 августа` | [07-01 → 08-31] |
| `Белые ночи с 5 июня по 6 июля` | [06-05 → 07-06] |
| `1 ноября` | [11-01 → 12-31] |
| пустая строка / `весь сезон` | [01-01 → 12-31] |

### Алгоритм парсинга

1. Очистка текста (`<br>`, `\r\n` → пробелы)
2. Удаление текстовых названий сезонов ("Белые ночи с 5 июня..." → "с 5 июня...")
3. Разбивка по "и" / ","
4. Для каждой части — регулярные выражения:
   - `с/со DD месяца по/до DD месяца` → диапазон
   - `до/по DD месяца` → от начала года
   - `с/со/от DD месяца` → до конца года
   - `DD месяца по DD месяца` (без "с") → диапазон
   - `DD месяца` (голая дата) → до конца года

### Нормализация времени

```
"10.00 - 18.00" → ("10:00", "18:00")
"6.00"          → "06:00"
```

### Нормализация дней недели

```
"Пт  - Сб" → "Пт-Сб"
"Вс - Чт"  → "Вс-Чт"
```

## WordPress PHP-сниппет

Установлен через плагин WPCode. Endpoint: `GET /wp-json/navibot/v1/prices`

Читает:
- Все WooCommerce-товары (`post_type => 'product'`)
- ACF-поля ценовых таблиц (b0-d0 сезоны, b1-d1 даты, b2-g2 строки с ценами)

## Процесс синхронизации (app.py: `sync_from_wp`)

1. `GET https://teplohod-restoran.ru/wp-json/navibot/v1/prices`
2. Для каждого теплохода из ответа:
   - Ищем в БД по имени (`get_boat_by_name`)
   - Если не найден — создаём новый (`create_boat`)
   - Парсим цены через `parse_wp_boat()`
   - Заменяем все цены в БД (`replace_prices_for_boat`)
3. Логируем результат (`log_sync`)

## Добавление нового теплохода на сайт

Если на сайте появился новый теплоход:
1. При синхронизации NaviBot автоматически создаст его в БД
2. Цены загрузятся из WP
3. Причал, стоимость уборки — нужно заполнить вручную (Админ → Теплоходы → Изменить)
